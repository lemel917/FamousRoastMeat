# 食材開關與菜單微動效 — 設計文件

日期：2026-07-18
狀態：已與需求方確認定案
前置：延續《2026-07-18-電子菜單-design.md》已上線的系統（GitHub Pages + Firebase Realtime Database）

## 目標

1. **減少後台開關**：便當區從 19 個品項開關改為 6 個「食材」開關（叉燒、燒肉、燒鴨、油雞、鴨腿飯、臘腸）；拼盤區 11 個開關不變。後台合計 30 → 17 個開關。
2. **菜單頁微動效**：保持現有復古紙感版面，加入精緻微動效（進場、懸停、蓋章）。

- 19 個便當品項（b01–b19）全數被覆蓋。
- 需求方確認的邊界：燒鴨開關**不**連動拼盤的燒鴨 1/4隻/半隻/一隻（p04–p06）；鴨腿飯開關**不**連動鴨腿一隻（p10）；這些由拼盤區自己的開關控制。

### 賣完換算規則

```
品項賣完 = 管到它的任一食材開關賣完 OR soldOut[品項id] 存在
```

- 共管品項（如三寶飯屬叉燒＋燒鴨＋油雞）：任一食材賣完即標賣完；只有全部相關食材恢復才恢復。
- 拼盤 p07/p08/p09 為雙重控制：食材開關或自身開關任一賣完即標。
- 換算在**讀取端**進行（菜單頁與後台頁各自從快照換算），資料庫不存品項層級的派生結果。

```
const INGREDIENTS = [
  { id: "chashao", name: "叉燒", items: ["b01","b03","b04","b05","b06","b09","b16","b18","p07"] },
  { id: "shaorou", name: "燒肉", items: ["b06","b08","b09","b10","b11","b12","p08"] },
  { id: "shaoya",  name: "燒鴨", items: ["b02","b03","b05","b06","b07","b11","b14","b18"] },
  { id: "youji",   name: "油雞", items: ["b04","b05","b07","b10","b15","b17","b18","p09"] },
  { id: "yatui",   name: "鴨腿飯", items: ["b19"] },
  { id: "lachang", name: "臘腸", items: ["b12","b13","b14","b15","b16"] },

```

`MENU_DATA` 本身（品名、價格、標籤）不變。

### Firebase

- 路徑不變（`soldOut/{key}`），key 的值域改為：6 個食材 id ＋ 11 個拼盤品項 id（p01–p11）。
- 值一律 `true`；恢復供應＝刪除節點（沿用既有語意，絕不寫 `false`）。
- 資料庫規則不變。
- **舊資料遷移**：不需要程式遷移。部署後在後台按一次「全部恢復供應」清空舊的 b01 式資料即可。殘留舊鍵對新版頁面無害（b01 不在任何換算來源中會被忽略）；新鍵對還沒更新快取的舊版頁面也無害（找不到對應 data-id 而忽略）。

## 後台頁改動（admin.html / js/admin.js / css/admin.css）

- 便當區標題改為「便當（食材開關）」，呈現 6 個開關；每個開關下方以灰色小字列出連動品項名稱（例：「影響 9 項：叉燒飯、叉鴨飯、⋯」），文字由 `INGREDIENTS` 與 `MENU_DATA` 自動產生，不手寫。
- 拼盤區維持 11 個品項開關。
- 開關狀態仍一律以 `onValue` 快照為準；寫入失敗彈回＋alert；「全部恢復供應」行為不變（`remove(soldOutRef)` 清空整包）。
- 登入、登出、錯誤處理（含 SDK 動態載入失敗）、RWD 全部沿用，不動。

## 菜單頁改動

### 換算邏輯（js/menu.js）

`applySoldOut(soldOut)` 改為：先由 `INGREDIENTS` 展開食材賣完的品項集合，聯集 `soldOut` 中直接命中的品項 id，再切換各列的 `.sold-out`。函式介面（輸入快照物件、操作 DOM class）不變。

### 微動效（css/menu.css ＋ 少量 js/menu.js 觸發）

1. **進場動畫**：頁面載入後，分類標頭先淡入，品項列自上而下依序「淡入＋上浮 8px」，每列間隔約 40ms（CSS animation ＋ `animation-delay` 以 CSS 變數或 inline style 依索引設定）。
2. **懸停/觸碰**：品項列 hover 時微微抬起（`translateY(-2px)`）＋淡陰影，transition 約 150ms。
3. **蓋章動畫**：品項「變為賣完」時，「已賣完」印章以蓋章動效落下——縮放 1.6→1、旋轉約 -8deg 定格、落定瞬間輕微震動感（keyframes），整列變灰與價格劃線以 300ms transition 平滑過渡。恢復供應時印章淡出。首次載入即為賣完的品項不播蓋章動畫（直接呈現最終狀態），避免開頁一排章亂蓋。
4. **`prefers-reduced-motion: reduce`**：停用以上所有動畫與過渡（媒體查詢一次關閉）。
5. 純 CSS transition/animation，不引入任何動畫函式庫；JS 只負責在狀態切換時加/移 class。

## 錯誤處理

全部沿用現有機制：SDK 動態載入失敗與 onValue 錯誤顯示同步失敗橫幅（含「請重新整理頁面」）、菜單離線仍可顯示、後台寫入失敗彈回。本次不新增錯誤路徑。

## 測試重點

1. 共管品項換算：叉燒賣完 → 三寶飯/招牌飯等變灰；叉燒恢復但燒鴨仍賣完 → 三寶飯仍灰、叉燒飯恢復。
2. 雙重控制：油雞賣完 → p09 油雞腿一隻變灰；油雞恢復、但 p09 自身開關賣完 → p09 仍灰。
3. 臘腸開關 → 臘味飯等 5 項連動。
4. 後台 17 個開關與連動品項小字正確；全部恢復供應清空一切。
5. 跨頁即時同步不重整。
6. 蓋章動畫、hover、進場 stagger；`prefers-reduced-motion` 模擬下全部靜止。
7. 手機寬度 RWD 不回歸。
