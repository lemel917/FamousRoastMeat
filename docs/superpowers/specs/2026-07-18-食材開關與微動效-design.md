### 賣完換算規則

```
品項賣完 = 管到它的任一食材開關賣完 OR soldOut[品項id] 存在
```

- 共管品項（如三寶飯屬叉燒＋燒鴨＋油雞）：任一食材賣完即標賣完；全部相關食材恢復才恢復。
- 食材開關只管飯類主餐；拼盤與單點（含 p07 叉燒一份、p08 燒肉一份、p09 油雞腿一隻）不受食材開關連動，一律由其自身開關控制。
- 換算在讀取端進行（兩頁各自從快照換算），資料庫不存品項層級的派生結果。

## 菜單頁視覺（照手寫臘版圖重刻）

- **整體**：淡米色紙面（比現版更淡、更乾淨），**無外框**；版心置中，桌機寬約 1400px。
- **頁首**：置中特大書法「名門燒臘」；左右兩側各一組金棕色雲紋裝飾＋細橫線延伸至邊緣。移除英文副標與右上標語。
- **分類標題**：手寫書法風「飯類主餐」「燒臘拼盤與單點」，字底襯一抹金棕色墨刷筆觸（CSS 漸層或內嵌 SVG 筆刷形狀），左對齊於各自欄位頂端。
- **版面**：桌機左右兩大欄（飯類主餐佔左、約 55–60%；拼盤與單點佔右），中間細直線分隔。飯類內部再分兩子欄（11＋9 項），各子欄有「一般｜加肉」標頭；拼盤與單點內部也分兩子欄（圖上為 6＋7 配置，紅茶/例湯前有一條實線分隔作「飲品湯品」視覺區隔）。
- **品項列**：品名黑色粗體楷/明體感、價格紅色粗體；列與列之間用**虛線**（dotted）分隔，取代現在的實線；份量備註（三人份等）小字灰棕色。
- **字體**：沿用 Noto Serif TC；書法標題另掛一款繁體可用的書法/毛筆風 Web 字型（實作時挑 Google Fonts 或 CDN 上可用者，僅標題與分類標題使用，需設 `font-display: swap` 與 serif 後備）。
- **頁尾**：移除「現點現切・外帶歡迎」標語（圖上沒有，完全照圖）。
- **RWD**：平板直式/手機單欄堆疊（飯類主餐 → 拼盤與單點），子欄合併為單欄；價格靠右不換行；斷點沿用 960px/600px。
- **賣完呈現**：沿用「整列變灰＋價格劃線＋紅色已賣完印章」，配新版視覺微調配色。

## 菜單頁微動效

1. **進場**：分類標頭先淡入，品項列自上而下依序「淡入＋上浮 8px」，每列間隔約 40ms。
2. **懸停/觸碰**：品項列微抬（translateY(-2px)）＋淡陰影，150ms transition。
3. **蓋章**：品項變為賣完時，印章縮放 1.6→1＋旋轉 -8deg 落下、落定微震；整列變灰 300ms 過渡；恢復時印章淡出。首次載入即賣完的品項不播蓋章動畫。
4. **`prefers-reduced-motion: reduce`**：全部動畫與過渡停用。
5. 純 CSS transition/animation；JS 只負責切換 class。

## 後台頁改動

- 飯類區標題「飯類主餐（食材開關）」，6 個開關；每個開關下方灰色小字列出連動品項（由 `INGREDIENTS`＋`MENU_DATA` 自動產生）。
- 拼盤與單點區 13 個品項開關。
- 自選三寶飯不出現在後台（無開關可控）。
- 開關以 `onValue` 快照為準、寫入失敗彈回＋alert、「全部恢復供應」清空整包——皆沿用。
- 登入、錯誤處理、RWD 沿用不動。後台視覺不改版（功能導向）。

## 錯誤處理

全部沿用現有機制（SDK 動態載入、同步失敗橫幅含「請重新整理頁面」、離線菜單照常顯示、寫入失敗彈回）。本次不新增錯誤路徑。

## 測試重點

1. 共管品項：叉燒賣完 → 三寶飯/招牌飯等灰；叉燒恢復但燒鴨仍賣完 → 三寶飯仍灰、叉燒飯恢復。
2. 不連動拼盤：油雞賣完 → p09 不灰（僅飯類受影響）；p09 自身開關賣完 → p09 灰。
3. 臘腸開關 → 5 項連動；b20 自選三寶飯在任何開關組合下都不會灰。
4. p12 紅茶、p13 例湯開關可獨立標賣完。
5. 後台 19 個開關與連動小字正確；全部恢復供應清空一切。
6. 新版視覺對照手寫臘版圖：置中書法標題＋雲紋、書法分類標題＋墨刷、虛線分隔、無外框；人氣/招牌徽章正常顯示（b01、b05、p01 人氣，b18 招牌）。
7. 蓋章/hover/進場動效；`prefers-reduced-motion` 下全靜止。
8. 跨頁即時同步不重整；手機寬度 RWD。
