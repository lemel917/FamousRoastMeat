# 名門燒臘電子菜單 — 設計文件

日期：2026-07-18
狀態：已與需求方確認定案

## 目標

做一個兩頁式網站：

1. **菜單頁**（`index.html`）：給客人看的電子菜單，內容與視覺依照《名門燒臘電子菜單_設計版.pdf》。
2. **後台頁**（`admin.html`）：老闆用來切換每道菜「賣完／供應中」；切換後菜單頁**自動**更新，不需重新整理。

部署在 **GitHub Pages 免費方案**（純靜態），賣完狀態存 **Firebase Realtime Database 免費 Spark 方案**。菜單頁與後台頁會在不同裝置上使用（客人手機、老闆手機/電腦）。

## 架構總覽

```
GitHub Pages（免費靜態網站，公開 repo）
├── index.html    ← 菜單頁
├── admin.html    ← 後台頁（前端帳密檢查）
├── menu-data.js  ← 菜單品項資料（名稱、價格、分類、標籤）
├── css/          ← 樣式（兩頁共用底層 + 各自頁面樣式）
└── js/           ← 各頁邏輯 + Firebase 初始化設定

Firebase Realtime Database
└── soldOut/{品項編號}: true   ← 只存「賣完」狀態，別的都不存
```

- 菜單品項資料寫死在 `menu-data.js`。改菜單＝改這個檔案、推上 GitHub。後台**不能**新增/編輯品項（範圍外）。
- Firebase 只存賣完狀態。品項不存在於 `soldOut` 底下即代表供應中。
- 兩頁都用 Firebase SDK 的 `onValue` 即時監聽：後台一切換，菜單頁一秒內自動反映。
- Firebase SDK 以 CDN `<script type="module">` 方式載入，不需要打包工具、不需要 Node.js。

## 資料設計

### menu-data.js

每個品項有固定編號：便當 `b01`–`b19`、拼盤 `p01`–`p11`。內容完全照 PDF：

- **便當**（19 項）：`{ id, name, price, priceExtra, tag }`
  - `price` 一般價、`priceExtra` 加肉價。
  - `tag`：`人氣`（叉燒飯、三寶飯）、`招牌`（招牌飯），多數品項無標籤。
- **燒臘拼盤**（11 項）：`{ id, name, price, note, tag }`
  - 單一價格；`note` 為份量備註（如「三人份」「六人份」），無備註者省略。
  - `tag`：`人氣`（美味雙拼）。

### Firebase

```
soldOut/
  b01: true      ← 叉燒飯賣完
  p03: true      ← 美味四拼賣完
```

- 路徑只有一層，值一律 `true`；恢復供應＝刪除該節點。
- 資料庫規則：`soldOut` 開放讀寫（`.read: true, .write: true`），其餘路徑全部拒絕。

## 菜單頁（index.html）

### 視覺

照 PDF 設計版重現：米色底、棕色細外框、書法感「名門燒臘」標題與英文副標、「飯／便當」「盤／燒臘拼盤」分區標頭、紅色價格、「人氣」「招牌」小徽章、便當區的「一般｜加肉」欄位標頭、底部「現點現切・外帶歡迎」。中文字體用系統字體堆疊＋Google Fonts 的書法/明體風格字型（僅標題）。

### 賣完呈現

- 品項整列變灰、價格加刪除線、蓋上紅色「已賣完」印章風格標籤。
- 狀態透過 `onValue` 即時更新，無需重整、無彈窗。

### RWD

- 電腦／平板橫式：兩欄版面（便當｜拼盤），貼近 PDF 構圖。
- 平板直式／手機：單欄直式捲動，字級與行距放大；價格靠右對齊、不換行。

## 後台頁（admin.html)

### 登入（前端帳密檢查）

- 開啟先顯示置中登入表單（帳號＋密碼）。
- 帳密**寫死在 JS 檔最上方**兩個常數，方便自行修改；預設 `admin` / 部署前自訂。
- 登入成功記在 `localStorage`，重開瀏覽器免重登；提供「登出」按鈕。
- 帳密錯誤顯示中文提示。
- **已知限制（需求方知情接受）**：帳密在公開 repo 的原始碼中可見，資料庫亦開放寫入；此登入僅防路人誤入，不是真正的資安防護。未來若需要，可升級 Firebase Authentication，架構不用改。

### 功能

- 依分類列出全部 30 個品項，每列一個「供應中／已賣完」開關。
- 切換即寫入 Firebase（賣完＝寫入 `true`；恢復＝刪節點），開關狀態同樣走 `onValue` 監聽，多裝置同時開後台也會同步。
- 「**全部恢復供應**」按鈕：需二次確認，一鍵清空 `soldOut/`，供每日開店重置。

### RWD

- 手機：開關加大好按、列高足夠防誤觸（老闆多半用手機操作）。
- 電腦：清單呈兩欄。登入表單各種螢幕皆置中。

## 錯誤處理

- **菜單頁連不上 Firebase**：菜單照常顯示（品項資料在本地檔），頂端顯示小提示「賣完狀態同步中，請稍候」；連上後提示消失。
- **後台寫入失敗**：開關彈回原狀，顯示錯誤訊息。
- **登入失敗**：中文錯誤提示，不鎖定、不限次數（練習專案）。

## 測試

- 本機以簡單靜態伺服器（如 `python -m http.server` 或 VS Code Live Server）開兩個視窗，驗證：後台切換 → 菜單頁即時變灰；全部恢復；斷網提示。
- 手機/平板寬度以瀏覽器裝置模擬驗證 RWD 斷點。
- 部署後在真實手機驗證一次。
